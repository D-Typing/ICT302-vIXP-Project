Overview

The virtual Internet Exchange Point (vIXP) is a simulated network fabric designed for educational and research purposes. It allows participants to establish BGP peering sessions and exchange routing information in a controlled environment.

This is a virtual IXP. No real internet traffic is exchanged. All sessions are confined to the vIXP fabric.

Requirements

To connect to the vIXP, participants must meet the following requirements:

  • A valid Autonomous System Number (ASN)
  • A router capable of BGP-4 and MP-BGP
  • Registered IRR route objects for all announced prefixes
  • A published peering policy on PeeringDB
  • A 24/7 reachable NOC contact

BGP Configuration

Configure your BGP session to the route servers using the following details:

ASNAS65000AS65000
IPv4 Address192.0.2.1192.0.2.2
IPv6 Address2001:db8::12001:db8::2
MD5 AuthOptionalOptional

Example Cisco IOS configuration:

router bgp 64500
 neighbor 192.0.2.1 remote-as 65000
 neighbor 192.0.2.1 description vIXP-RS1
 neighbor 192.0.2.1 soft-reconfiguration inbound
 !
 address-family ipv4
  neighbor 192.0.2.1 activate
  neighbor 192.0.2.1 prefix-list ANNOUNCE out
 exit-address-family

Peering Policy

The vIXP operates an open peering policy. All members are encouraged to peer with each other via the route servers. Bilateral sessions are also permitted.

Members must not point a default route toward the route servers. Violation will result in session termination.
